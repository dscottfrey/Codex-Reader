## A1. Inline Image Zoom — Lightbox

Epub books frequently contain inline images: illustrations, diagrams, maps, photographs. They render naturally within the CSS Columns layout and are baked into the pre-rendered page UIImage. When the user wants a closer look, Codex provides a lightbox zoom interaction that keeps the book in view and requires no navigation.

**What Apple Books does wrong:** tapping an image in Books pushes a new full-screen view controller with a navigation bar and a back button. The image appears detached from its context; the reader feels like they have left the book. Dismissal is a navigation pop. The whole pattern implies the image is a separate destination rather than part of the page being read.

**What Codex does instead:** the image expands in place over the page, the book remains visible behind a dark overlay, and dismissal returns the image to exactly where it came from. The reader never loses their place or their sense of where the image sits in the text.

### Image Tap Detection

Since pages are pre-rendered UIImages, taps on the display layer land on a flat bitmap with no element awareness. The WKWebView interaction layer (§3.6 of `01_RENDERING_ENGINE.md`) solves this. Every tap on the reading surface is first checked against the interaction WebView before being handled as a chrome-toggle or page-turn:

```javascript
// Called via evaluateJavaScript on every tap before other tap handling.
// Returns image info if an <img> element (or its containing <figure>) is at
// the tap coordinates, null otherwise.
function imageAtPoint(x, y) {
    const el = document.elementFromPoint(x, y);
    const img = el?.tagName === 'IMG' ? el
              : el?.closest('figure')?.querySelector('img');
    if (!img) return null;
    const rect = img.getBoundingClientRect();
    return {
        src:     img.src,             // http://localhost:[r2port]/... via R2Streamer
        x:       rect.left,
        y:       rect.top,
        width:   rect.width,
        height:  rect.height,
        caption: img.closest('figure')
                    ?.querySelector('figcaption')
                    ?.textContent?.trim() ?? null
    };
}
```

If this returns a result, the tap is consumed as an image tap and the lightbox is presented. If it returns null, normal tap handling continues (chrome toggle or page turn).

The returned `rect` is in WKWebView coordinate space. Convert to screen coordinates using the interaction WebView's frame before using as the hero animation launch frame.

### Lightbox Presentation

1. **Hero launch.** The image animates outward from its position in the page — expanding and moving to fill a large portion of the screen — over a dark overlay that fades in simultaneously. The animation origin is the image's screen-space rect from the hit-test. The reader's eye follows the image as it grows; spatial context is preserved.

2. **Display.** The image fills most of the screen, centered, with comfortable insets. If the epub provides a `<figcaption>`, it appears as small text below the image. No navigation bar. No close button. No chrome beyond the image and optional caption.

3. **Zoom and pan.** Pinch to zoom, with momentum and rubber-band bounce at edges — identical feel to Photos.app. Double-tap toggles between fitted and 100% size.

4. **Dismiss.** The user swipes down; the image tracks the finger and, on release, flies back to its exact origin position in the page with the overlay fading out. Alternatively, a tap on the dark overlay outside the image dismisses with the same return animation. No button needed — both gestures are universal. The book reappears exactly as it was.

**Image loading:** the image `src` is an `http://localhost` URL served by R2Streamer. Load it directly into a `UIImageView` — no extraction or temp-file handling required.

### Caption Handling

Show the `<figcaption>` text if present. Use a small, legible secondary font — not the user's chosen reading font. Show nothing if no caption exists. Do not synthesise a caption from `alt` text, filename, or any other source.

### What Is Explicitly Not Here

**No gallery mode.** There is no swipe-left/right to browse all images in the chapter from within the lightbox. Gallery mode severs the image from the text surrounding it — the reader loses the context of why the image is there. The lightbox is intentionally single-image: the reader looks at one image, returns to the page, continues reading. This is a firm design decision, not a deferred feature.

### Interaction WebView During Lightbox

The UIImageView display layer remains visible (dimmed by the overlay) while the lightbox is open. The interaction WebView is not active during lightbox display — the user is viewing an image, not reading text. Restore normal interaction layer behaviour on lightbox dismissal.
